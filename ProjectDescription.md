# Introduction

The goal of this project is to create a fake news prediction system. Fake news is a major problem that can have serious negative effects on how people understand the world around them. You will work with a dataset containing real and fake news in order to train a simple and a more advanced classifier to solve this problem. This project covers the full Data Science pipeline, from data processing, to modelling, to visualization and interpretation.

## Practicalities

**Deadline:** Your report is due by **16:00 on Friday 27 March**, and it must be submitted via Digital Exam. Make sure that you submit as a group. In order to do this, you will need to create your group under the "People" section. Create these groups well before the project deadline so any technical problems can be dealt with early. Please let us know (by starting a discussion in Absalon) if something is not working.

**Time Management:** Each part of the project has a recommended date by which you should have most of the work completed. You may need to revisit some parts throughout the project as you develop more advanced models and need more features. Resist the temptation to attempt to complete the whole project in the final week.

**Length:** The final report should be handed in as a **PDF file**, with a maximum length of **6 pages** in the format defined below. The expected number of pages is noted in the header of each section. In addition to text, you may include tables, figures, and code, but you should not exceed the maximum length. Appendices are permitted, but are not guaranteed to be considered.

**Format:** The report must be written using the following LaTeX template: [arxiv/bio-arxiv Preprint Template](https://da.overleaf.com/latex/templates/style-and-template-for-preprints-arxiv-bio-arxiv/fxsnsrzpnvwc). Structure your report so that section numbers correspond to the parts. You do not need to write an introduction or an abstract. The short format means you will need to prioritize what you include — put the most central observations in the main text, and refer to less important material in the appendix if necessary. The report should be well-organized, technically precise, and polished (e.g., typo-free).

**Contributions:** If tasks are distributed such that some group members are primarily responsible for particular parts of the report, please explicitly state so using percentages to specify each member's contribution. If no such annotation is provided, equal contribution from all members will be assumed. Please also state the **KUIDs** of all group members on the front page of the report.

**Code:** Your group should make their code available for examination, either by uploading it directly or by including a link to a GitHub repository in your report. Your project may have created large intermediate files that cannot easily be uploaded — your code should therefore include a `README` file that clearly explains how to run the code and reproduce the results claimed in your report.
